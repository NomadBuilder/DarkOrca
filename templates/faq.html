Frequently Asked Questions

Find answers to common questions about DarkOrca security assessment platform

General Questions

What is DarkOrca?
DarkOrca is a comprehensive security assessment platform designed for defensive security reconnaissance and risk scoring. It provides automated scanning capabilities to identify vulnerabilities, misconfigurations, and security weaknesses across web applications and services. DarkOrca combines both passive reconnaissance and active exploitation testing to deliver thorough security assessments with detailed risk analysis.
What types of vulnerabilities does DarkOrca detect?
DarkOrca can detect over 30 different types of security issues, including:
  • Web Application Vulnerabilities: XSS, SQL Injection, CSRF, SSRF, XXE, Command Injection, Path Traversal, File Upload vulnerabilities, IDOR, Template Injection
  • API Security: GraphQL vulnerabilities, JWT security issues, OAuth/SAML misconfigurations
  • Infrastructure Issues: SSL/TLS misconfigurations, missing security headers, DNS security problems, weak encryption
  • WordPress-Specific: Plugin/theme vulnerabilities, user enumeration, configuration issues
  • Information Disclosure: Exposed backup files, sensitive data leaks, error message disclosure
For a complete list, visit our Security Glossary.
Is DarkOrca free to use?
DarkOrca is open-source and available for use. However, some integrated tools (like WPScan) may require API tokens for full functionality. The core scanning capabilities are available without cost. Always check the licenses of integrated third-party tools.

Scan Modes

What are the different scan modes and when should I use them?
DarkOrca offers three scan modes:
  • Defensive Mode: Passive, read-only scanning that performs reconnaissance without attempting exploitation. Safe for production systems, perfect for initial assessments and compliance checks. Use this when you need to assess security without risking system stability.
  • Offensive Mode: Active scanning with exploitation attempts to confirm vulnerabilities. Should only be used with explicit authorization as it may impact system stability or trigger security alerts. Use this in staging environments or when you have explicit permission to test production systems.
  • Comprehensive Mode: Combines both defensive and offensive testing for the most thorough assessment. Recommended for complete security evaluations in authorized testing environments.
Is it safe to use Offensive Mode on production systems?
No. Offensive Mode attempts actual exploitation of vulnerabilities, which can:
  • Impact system stability and performance
  • Trigger security alerts and intrusion detection systems
  • Potentially cause data corruption or service disruption
  • Create false security incidents
Always use Offensive Mode only in staging/test environments or with explicit written authorization for production systems. Defensive Mode is designed to be safe for production use.
What is Exhaustive Mode?
Exhaustive Mode performs more thorough scanning by using extended payload sets and testing more parameters. This results in more comprehensive coverage but takes significantly longer to complete. Use Exhaustive Mode when you need the most thorough assessment possible, but be aware that scans may take several hours depending on the target size.

Authorization & Legal

Do I need authorization to use DarkOrca?
Yes, always. You must only use DarkOrca on systems you own or have explicit written authorization to test. Unauthorized scanning is illegal in most jurisdictions and may result in:
  • Criminal charges under computer fraud and abuse laws
  • Civil liability for damages
  • Professional consequences
Even Defensive Mode scanning without authorization may violate terms of service and local laws. Always obtain proper authorization before conducting security assessments.
What constitutes proper authorization?
Proper authorization typically includes:
  • Written permission from the system owner or authorized representative
  • Scope definition - what systems, IPs, or domains are covered
  • Time frame - when the testing is authorized
  • Method authorization - which scan modes and techniques are permitted
  • Contact information for the authorizing party
For production systems, always get explicit written authorization. For systems you own or control, ensure you have the legal right to test them.

Results & Reports

How do I interpret the risk score?
DarkOrca provides a risk score from 0-100, calculated using a weighted algorithm:
  • 0-10: Minimal risk - mostly informational findings
  • 11-25: Low risk - minor issues that should be addressed
  • 26-50: Medium risk - moderate security concerns requiring attention
  • 51-75: High risk - serious vulnerabilities that should be prioritized
  • 76-100: Critical risk - immediate threats requiring urgent remediation
The score also includes category-specific breakdowns to help prioritize different types of vulnerabilities. Always review individual findings, as the score is a guideline and should not replace professional judgment.
What file formats can I export results to?
DarkOrca supports multiple export formats:
  • JSON: Machine-readable format for programmatic analysis and integration
  • Markdown: Human-readable format perfect for documentation and reports
  • PDF: Professional formatted report with optional AI-generated executive summary
All formats include the complete scan results, findings, evidence, and remediation guidance.
How long are scan results stored?
In the web interface, scan results are available for immediate viewing after completion. Shareable result links remain active for 30 days. For long-term storage, download and save reports in your preferred format (JSON, Markdown, or PDF). Active scans remain in memory for status checks for up to 1 hour after completion.
What is the AI-generated executive summary?
The AI-generated executive summary provides a high-level analysis of scan results using AI models (OpenAI GPT or Anthropic Claude). This feature requires an API key and is optional. The summary offers:
  • Executive-level overview of security posture
  • Priority recommendations for remediation
  • Risk assessment summary
  • Business impact analysis
Configure your API key in environment variables to enable this feature. The summary is only included in PDF reports.

Accuracy & False Positives

How accurate are DarkOrca's findings?
DarkOrca uses industry-standard scanning techniques and integrates proven tools like SQLMap, Nuclei, and Nmap. However, automated scanning tools can produce false positives. DarkOrca includes evidence collection to help verify findings, including HTTP requests/responses and exploitation details where applicable. Always manually verify critical findings before taking remediation actions, especially in Offensive Mode where exploitation attempts provide stronger confirmation.
What should I do if I think a finding is a false positive?
If you suspect a false positive:
  1. Review the evidence provided with the finding, including request/response data
  2. Manually verify the vulnerability using the provided payloads or techniques
  3. Check if the finding was from Offensive Mode (more reliable) or Defensive Mode (may need verification)
  4. Consider context - some findings may be technically accurate but low-risk in your specific environment
  5. Document your findings if you determine it's a false positive for future reference
Remember that false negatives (missed vulnerabilities) are also possible - no scanner catches everything.

Performance & Limitations

How long do scans take?
Scan duration varies significantly based on:
  • Scan mode: Defensive is faster, Offensive/Comprehensive takes longer
  • Target complexity: Number of endpoints, technologies, and applications
  • Exhaustive mode: Adds significant time but provides more thorough coverage
  • Network speed: Latency and bandwidth to the target
  • Rate limiting: Target's rate limiting can slow scans
Typical scans range from 5-30 minutes for simple targets to several hours for complex applications in Exhaustive Mode. The web interface provides real-time progress updates and time estimates.
What are DarkOrca's limitations?
DarkOrca has several limitations to be aware of:
  • Automated scanning cannot replace manual testing - Complex business logic flaws may be missed
  • Authentication-dependent testing is limited - Many scanners work best on public-facing endpoints
  • False positives/negatives are possible with any automated tool
  • Rate limiting and WAFs may prevent complete scanning
  • Some vulnerabilities require context that automated tools cannot understand
  • Network access required - Cannot test internal-only systems without network access
Use DarkOrca as part of a comprehensive security program that includes manual testing, code review, and other assessment methods.

Setup & Configuration

What tools does DarkOrca require to be installed?
DarkOrca integrates with several tools, but many features work without external dependencies:
  • Python 3.8+ - Required for core functionality
  • SQLMap - Optional, for SQL injection testing (requires Python 2/3)
  • Nuclei - Optional, for vulnerability template scanning (requires Go)
  • Nmap - Optional, for network scanning
  • WPScan - Optional, for WordPress-specific scanning (requires Ruby)
  • Subfinder/Amass - Optional, for subdomain enumeration
Core scanners work without external tools, but you'll get more comprehensive results with integrated tools installed.
How do I configure email notifications?
Email notifications use Resend (recommended) or SMTP. Configure in your .env file:
  • Resend (Recommended): Set RESEND_API_KEY, FROM_EMAIL, FROM_NAME
  • SMTP (Fallback): Set SMTP_SERVER, SMTP_PORT, SMTP_USERNAME, SMTP_PASSWORD
  • Base URL: Set BASE_URL for proper links in emails
When starting a scan via the web interface, enter your email address to receive notifications when the scan completes.
Can I use DarkOrca from the command line?
Yes! DarkOrca provides a CLI interface. Basic usage: 
python cli.py https://example.com --format json --output report.json
See python cli.py --help for all available options including scan modes, scanner selection, and output formats.
